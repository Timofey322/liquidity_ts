# F80 v2

## Для чего проект

Это **исследовательская среда на Python** вокруг рынков **FX и металлов** (евро, фунт, золото и сходные инструменты). Она объединяет в одном репозитории то, что в реальной работе обычно разъезжается по разным репам: **получение и проверка данных**, **рыночная логика и фичи**, **обучение и бэктест ML- и rule-based стратегий**, а отдельно — **учёт сделок и «закрытие периода»** с отчётами и веб-интерфейсом.

Цель не в том, чтобы «продать готового робота», а в том, чтобы **прозрачно пройти путь от сырых котировок до измеримого результата** (сделки, кривая эквити, метрики) и при необходимости **свести это с бухгалтерским контуром**: нормализованный журнал, позиции, реализованный/нереализованный PnL, сверка с внешним снимком, KPI и markdown/json-отчёты.

## В чём идея

1. **Единый источник правды по данным.** Тики (например, Dukascopy) или готовые свечи (OANDA, CSV брокера) приводятся к одному формату, валидируются, на их основе строятся бары и датасеты для ML. Так меньше скрытых расхождений между «тем, на чём искали альфу», и «тем, что реально можно торговать».

2. **Два контура смысла: исследование и учёт.**  
   - **Исследование** — фракталы, зоны имбалансов (FVG), футпринт/профиль, сессионные признаки, Optuna + бустинги, отдельные research-модули (Session-POI, weekly XGB и т.д.). Здесь важны воспроизводимость, сплиты по времени, диагностика переобучения и калибровки.  
   - **Учёт** — из журнала сделок (в т.ч. экспортированного из бэктеста) считаются позиции, денежные потоки, PnL и сверка; результат — файлы отчётности и дашборд, удобный для повторных прогонов.

3. **Скорость там, где это больно.** Тяжёлые циклы по рыночной микроструктуре (фракталы, FVG, агрегация футпринта) вынесены в **Cython**; если среда без компилятора — те же API доступны через **NumPy-фолбэки**, чтобы код не ломался и идеи можно было проверять на меньших объёмах данных.

4. **CLI + тесты.** Основные сценарии доступны из командной строки (`python -m src.cli …`), критичные куски учёта и важность признаков покрыты **pytest** — проект рассчитан на то, что его можно сопровождать и развивать, а не только «запускать раз в месяц».

Итого: **идея** — связать **генерацию сигналов и оценку стратегии** с **аккуратным постинвестиционным учётом** в одной кодовой базе, с упором на воспроизводимость и измеримость, а не на маркетинговые обещания.

## Что внутри (логика, не список файлов)

- **Данные и валидация** — загрузка тиков/свечей, нормализация, проверка OHLC и временных рядов.  
- **Рыночная логика** — уровни, поведение цены, объёмно-ценовые конструкции; подготовка признаков под ML.  
- **ML и research** — датасеты с горизонтом и порогами доходности, обучение с подбором гиперпараметров (в т.ч. Optuna), бэктест с учётом комиссии; отдельные экспериментальные контуры (weekly XGB, session-ориентированные стратегии).  
- **Учёт** — нормализация сделок, проводки по позициям, отчёт о закрытии периода, KPI; Streamlit-приложение для просмотра и пересчёта без ручной сборки скриптов.

Подробные **команды по контурам**, пути артефактов и замечания по данным — в **[PROJECT.md](PROJECT.md)**.

## Быстрый старт

```bash
pip install -r requirements.txt
python setup.py build_ext --inplace
```

Без компилятора C достаточно зависимостей: используются фолбэки в `src/fast/*_py.py`.

```bash
streamlit run tools/accounting/app.py
pytest -q
```

Остальные команды (закрытие периода, конвертация журналов, портфель из трёх инструментов, train/backtest) — в [PROJECT.md](PROJECT.md).
